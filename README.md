# DWM
(DeepSeek-based Words Memorizor)基于ds的背单词软件.

## 导入单词
- 使用UTF-8格式的.csv存储单词表. 
- 文件必须是vocab.csv, 放在主目录下
- 必须要有一个表头"word,meaning"
- 下面部分用逗号分隔, 左边英文词, 右边写中文(也可以包括词性, 多个示意, 甚至音标, 但是就是不能有逗号)
例如:
```csv
word,meaning
abandon,v. 抛弃, 放弃 离弃(家园/ 船只/ 飞机等) 遗弃(妻/ 子女等)
```

## 设置
设置是settings.json. 运行init.py可以自动创建
- "daily_new_words" 表示一日新学的单词数
- "daily_review_words" 表示一日复习的单词数
- "review_words_growth_days" 表示过多少天复习单词数就会+1, 这一点会直接更新上面那一条设置(赤石)
- "forgetting_curve_halflife" 单词记忆的半衰期(你觉得你过多久就会只记得一半单词, 记性越好数值越大, 推荐3.0)
- "halflife_growth_factor" 单词复习次数越多, 半衰期就越长. 公式为```基础值*(1+k*复习次数)```, k即此常数. 越大复习越有效, 推荐0.5. 更大的值倾向于让每天的复习故事包含更新的词. 而偏小的值可能让复习故事一直展示早期学习的词.
  - 这一项默认是可选的, 如果不填写, 会添加默认值0.5.

## 用户数据的结构
用户数据存储在user_data.json. 运行init.py可以自动创建
- current_day表示当前天数, 用于结合半衰期计算每个词的忘记概率.
- learned_words字段是一个字典, 里面装有所有学过的词(从单词表中抽取过的词)
  - 键是单词(不包括释义)
  - 值字典包含两个字段: "last_review"和"recognize_count"
    - last_review表示上次见到这个单词是位于第几天, "见到"包括新学, 以及复习时的抽取
    - recognized_count表示见过这个单词的次数. 当新学一个词语的时候, 会设置为1. 每次复习的抽取的时候, 会+1. 但是如果在新学单词过程中, 系统选择到了你已经掌握的单词, 那这个数值会被设置为1000, 表示"你从前就见过这个单词".

## 使用
首先, 按照[导入单词](#导入单词)的指引, 把单词表文件放在主目录下, 并重命名成vocab.csv.
然后, 运行一下init.py. 会生成settings.json. 根据[设置](#设置)部分的说明修改配置到自己喜欢的程度.
运行new_word.py可以学习新词, 运行prompt_generator可以进行复习(实际上是抽取一些需要复习的词, 然后生成一个可以丢给deepseek等GPT的提示词). 你可以在一天之中先复习, 再学习. 也可以反过来, 这取决于你. 一天的学习结束之后(或者一天的学习开始之前)要运行一下one_more_day.py, 来增加天数.
如果想要重置学习进度, 可以再次运行init.py, 会生成新的settings.json和user_data.json. 原本的两个文件会变成备份文件, 你需要自行处理.

## 提示与技巧
- 如果AI在复习词数特别多的情况下表现不佳, 可以选择将settings中的"daily_review_words"字段数值改小一点, 但是连续进行多次review. 尽管这样可能生成重复词语.
