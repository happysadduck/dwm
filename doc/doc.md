# 工作原理

## 如何结合GPT
我们穷人买不起好的电脑, 跑不动好的本地模型; 我们穷人还买不起API, 没法让程序去调用AI. 
但是实际上这都不是问题. 很多AI大模型允许你在网页上或者它们的应用程序上在聊天框里手动输入(或者粘贴)提示词来免费获取AI咨询. 大部分情况下, 调用API根本没有必要. 勤奋能够创造价值, 虽然程序不能自己去调用API, 但是程序可以告诉你提示词, 然后你就可以手动把提示词复制过去来问AI. 效果几乎没有差异. 
或者如果你是尊贵的富人, 你调用的起API或者跑得动本地模型, 那也只需要简单改造一下, 把提示词写进去就可以了. 
这就是prompt_generator为什么只生成提示词的原因.

## prompt_generator如何选择准备复习的单词

`prompt_generator.py` 的核心任务是从用户已学单词库中, 根据**遗忘曲线模型**和**个性化半衰期**, 动态挑选出当天需要复习的单词, 并生成一个包含这些单词的 AI 提示词. 整个过程分为以下几个步骤：

### 1. 数据准备
- 加载单词表 `vocab.csv`(所有可能的单词及其释义). 
- 加载配置文件 `settings.json`, 获取关键参数：
  - `daily_review_words`：基础每日复习单词数. 
  - `review_words_growth_days`：每过多少天, 复习单词数增加 1. 
  - `forgetting_curve_halflife`：基础半衰期(天). 
  - `halflife_growth_factor`：每次复习后, 半衰期增长系数. 
- 加载用户数据 `user_data.json`, 包含：
  - `current_day`：当前学习天数. 
  - `learned_words`：已学单词字典, 每个单词记录：
    - `last_review`：上次复习或出现的故事中的日期. 
    - `recognized_count`：该单词被认识的总次数(包括新词学习中的“认识”和故事中被选中复习的次数). 

### 2. 计算当日复习单词目标数量
- 初始基础值 = `daily_review_words`. 
- 额外增量 = `floor((current_day - 1) / review_words_growth_days)`. 
- 最终复习数量 = `min(基础值 + 增量, 单词表总大小)`.   
  例如：`daily_review_words=5`, `review_words_growth_days=3`, 则第1天=5, 第4天=6, 第7天=7... 直到达到上限(单词总数). 

### 3. 为每个已学单词计算遗忘概率(权重)
对于已学单词库中的每个单词, 计算其**个性化半衰期**和**遗忘概率**：

- **个性化半衰期**  
  `effective_halflife = base_halflife × (1 + growth_factor × recognized_count)`  
  其中 `base_halflife` 是配置值(如3天), `growth_factor` 是增长系数(如0.5).   
  - 一个单词被认识次数越多, 其半衰期越长, 遗忘越慢.   
  - 第一次认识(`recognized_count=1`)时, 半衰期为 `3×(1+0.5)=4.5`天；认识100次后, 半衰期长达 `3×(1+0.5×100)=153`天. 

- **距离上次复习的天数**  
  `delta_t = current_day - last_review`. 如果今天刚复习过(`delta_t=0`), 则遗忘概率设为1(实际会因半衰期长而权重仍然较低, 但保证它有机会被抽中). 

- **遗忘概率公式**  
  `p = 2 ^ (-delta_t / effective_halflife)`  
  这是一个指数衰减函数：当 `delta_t = effective_halflife` 时, `p=0.5`；当 `delta_t = 0` 时, `p=1`；当 `delta_t` 很大时, `p` 趋近于0.   
  概率越大, 表示该单词越可能被遗忘, 越需要复习. 

### 4. 加权随机抽样
- 将所有已学单词及其对应的遗忘概率 `p` 构成候选列表. 
- 使用 `random.choices(weights=p)` 按概率加权随机抽取 `N` 个单词(`N` 为当日复习目标数量).   
  **效果**：遗忘概率高的单词被抽中的概率也高, 但又不是绝对的“最可能遗忘的单词”一定会被选中, 保留了随机性, 避免重复复习相同的单词. 
- 如果抽取结果有重复(`k` 较大时可能), 会去重并补全不足的数量(从剩余单词中随机补充). 

### 5. 更新用户数据(关键！)
- 抽中的单词将在故事中出现, 这本身就是一次复习. 因此, 立即更新这些单词的记录：
  - `recognized_count += 1`(复习次数增加). 
  - `last_review = current_day`(将上次复习日期设为今天). 
- 保存更新后的 `user_data.json`.   
  **作用**：这些单词的半衰期会变长, 未来被抽中的概率下降, 从而实现“每个单词都有机会被复习”的均衡分布. 

### 6. 生成提示词
- 只提取选中单词的**英文**(不附带释义), 按列表格式拼接. 
- 使用固定提示词模板, 要求 AI 写一个包含所有这些单词的英文故事, 不限制长度, 不限制难度. 

### 为什么这个机制合理？

- **模拟真实遗忘**：基于艾宾浩斯遗忘曲线的指数衰减, 且复习次数影响半衰期, 符合“多次记忆后忘得慢”的规律. 
- **随机性避免枯燥**：不是简单地取遗忘概率最高的前 N 个, 而是加权随机, 让复习内容多样化. 
- **动态增长复习量**：随着学习天数增加, 复习单词数逐渐上升, 符合知识积累后需要更多维护的规律. 
- **自平衡**：每次被选中复习的单词会立即增加其 `recognized_count`, 降低未来选中概率, 确保长期学习效果. 

如果你在实现中需要调整参数(如 `halflife_growth_factor`), 可以通过修改 `settings.json` 来改变复习强度.
